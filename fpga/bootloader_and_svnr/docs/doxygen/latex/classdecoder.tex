\doxysubsection{AusfÃ¼hrliche Beschreibung}
Verarbeitet Bootloader Pakete und leitet diese an die entsprechenden Funktionalitäten des Bootloaders weiter. 

Bietet Signale zum Beschreiben des SVNR Rams ({\ttfamily o\+\_\+ram\+\_\+addr}, {\ttfamily o\+\_\+ram\+\_\+data}, {\ttfamily o\+\_\+ram\+\_\+wen}, {\ttfamily o\+\_\+ram\+\_\+runner\+\_\+begin}, {\ttfamily o\+\_\+ram\+\_\+uploading}) , sowie das {\ttfamily cpu\+\_\+en} Signal, welches die CPU des SVNRs steuern kann.

Auf jedes empfangene Bootloader Paket gibt der Decoder eine Response über {\ttfamily o\+\_\+status} an den UART TX weiter, welcher an die Host App zurück gesendet wird. Diese Response kann entweder als einfache Statusinformationen/\+Acknowledgment dienen, oder angeforderte Daten zurückgeben. ~\newline
 Die Hauptaufgabe des Decoders liegt in seiner Statemachine, in der die UART Inputs verarbeitet werden und entsprechende Aktionen in der Programmlogik des Bootloaders ausgeführt werden. Diese Logik umfasst\+:
\begin{DoxyItemize}
\item Korrektes Kopieren des Binärimage in den SVNR RAM
\item Prüfung der Integrität einer vollständigen Übertragung des Binärimage
\item Erzeugen von Statusmeldungen/\+Fehlermeldungen, welche als Response an die Host App zurück übertragen werden werden sollen
\item Stati zum Einhalten der Reihenfolge aller Debugger Funktionalitäten, hierzu gehört\+:
\item Breakpoints hinzufügen/löschen
\item SVNR CPU starten/stoppen 
\end{DoxyItemize}
