◆ bin2mnem()

int bin2mnem ( char * filename)

bin2mnem(char* filename): erstellt eine neue .txt Datei aus einer Binärdatei die mit Opcode und Adresse befüllt ist.

Im Parameter wird ein Dateipfad zu einer Datei angegeben. Diese wird geöffnet und gelesen. Anhand des Inhaltes werden die Positionen in dem Array des Typen cmd, der Opcode und die Adresse ermittelt. In einem Mapping wird nach zu dem Opcode korrespondierenden Mnemonics gesucht. Sollten diese gefunden werden, werden diese in dem Array mit der Adresse gespeichert. Darauf folgt das Speichern des erstellten Arrays in eine .txt Datei.

Noch zu erledigen

Im Moment gibt es zwei Mappings, wobei eins komplett ausreicht.

Funktion funktioniert noch nicht richtig. Es werden noch nicht alle Opcodes ausgelesen und die Datei wird fehlerhaft erstellt.

Parameter
filenamePfad zu einer Binäratei die Opcodes enthält und übersetzt werden soll.
Rückgabe
Gibt durch eine Zahl 1 oder 0 an ob die übersetzte Mnemonic-Datei erstellt wurde.

◆ formatFile()

void formatFile ( char * filename)

formatFile(char* filename): bringt eine Datei mit Assembler-Mnemonics in ein bestimmtes Format damit der Compiler benötigte Daten besser auslesen kann.

Im Parameter wird ein Dateipfad zu einer Datei angegeben. Diese wird geöffnet und gelesen. Jede Zeile der im Parameter übergebenen Datei wird in eine neue Datei geschrieben. Hierbei werden jedoch Leerzeilen, Leerzeichen sowie durch '#' oder ';' gekennzeichnete Kommentare entfernt. Die Methode bringt die Datei in ein bestimmtes Format, damit der Compiler die Daten besser auslesen kann.

Noch zu erledigen
Im Moment wird noch eine neue Datei erstellt, was ineffizient ist. Stattdessen könnte man den formatierten Text in einem Buffer speichern und den darin stehenden Inhalt dann kompilieren.
Parameter
filenamePfad zu einer Datei die Assembler-Mnemonics enthält und übersetzt werden soll.

◆ mnem2bin()

int mnem2bin ( char * filename)

mnem2bin(char* filename): erstellt eine neue .bin Datei aus einer Textdatei die mit Assembler Mnemonics befüllt ist.

Im Parameter wird ein Dateipfad zu einer Datei angegeben. Diese wird geöffnet und gelesen. Anhand des Inhaltes werden die Positionen in dem Array des Typen cmd, der Opcode und die Adresse ermittelt. Zuvor wird das gesamte Array mit dem Pakettypen 0x02 und 0x00 0x00 gefüllt. Dann wird in einem Mapping nach zu dem Mnemonic korrespondierenden Opcodes gesucht. Sollte diese gefunden werden, werden diese in dem Array mit der Adresse gespeichert. Darauf folgt das Speichern des erstellten Arrays in eine .bin Datei.

Noch zu erledigen
Adressen direkt beschreiben ohne Opcode.
Parameter
filenamePfad zu einer Datei die Assembler-Mnemonics enthält und übersetzt werden soll.
Rückgabe
Gibt durch eine Zahl 1 oder 0 an ob die übersetzte Binärdatei erstellt wurde.
